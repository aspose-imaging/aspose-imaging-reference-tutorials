## Common Issues and Solutions
- **Out‑of‑Memory Errors** – `ImageOptions` を使用してメモリ使用量を制限するか、ストリームで画像を処理してください。  
- **Loss of Metadata** – 変換時に `PreserveMetadata` を有効にして EXIF データを保持。  
- **Unexpected Color Shifts** – JPEG と PNG 間の変換時にカラープロファイルを確認してください。  

## Frequently Asked Questions

**Q: Can I compress images without losing quality?**  
A: Yes. Aspose.Imaging は PNG や TIFF などのロスレス圧縮と、JPEG などのロッシー形式向けに調整可能な品質設定を提供します。

**Q: How do I perform batch compression on thousands of images?**  
A: 「Batch Processing & Multi‑threading」チュートリアルを参照し、Java の `ExecutorService` を活用して最適なスループットを実現してください。

**Q: Is it possible to add a watermark before compression?**  
A: Absolutely. `Watermark` API でウォーターマークを適用し、その後圧縮処理を実行すれば、最終ファイルにウォーターマークが埋め込まれます。

**Q: Does Aspose.Imaging support DICOM image compression?**  
A: Yes. DICOM モジュールには医療画像規格に準拠した専用圧縮手法が含まれています。

**Q: What Java versions are compatible with Aspose.Imaging?**  
A: ライブラリは Java 8 以降、Java 11、Java 17 などの LTS バージョンをサポートしています。
