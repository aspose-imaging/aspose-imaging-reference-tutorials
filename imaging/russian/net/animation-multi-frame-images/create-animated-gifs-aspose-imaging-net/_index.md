### Руководство по внедрению

#### Обзор функции
Эта функция позволяет создавать GIF-файлы путем объединения нескольких кадров, что делает ее идеальной для анимации или последовательностей изображений.

##### Шаг 1: Определите каталоги
Начните с указания каталогов документов и выходных данных:

```csharp
string dataDir = "YOUR_DOCUMENT_DIRECTORY";
string outputDir = "YOUR_OUTPUT_DIRECTORY";
```

##### Шаг 2: Загрузка рамок
Загрузите все кадры изображений, которые вы хотите включить в свой GIF:

```csharp
private static IEnumerable<RasterImage> LoadFrames(string directory)
{
    foreach (var filePath in Directory.GetFiles(directory))
    {
        yield return (RasterImage)Image.Load(filePath);
    }
}
```
Здесь мы перебираем файлы в указанном каталоге и загружаем их как `RasterImage` объекты.

##### Шаг 3: Создайте GIF-файл
Инициализируйте свой GIF-файл первым кадром, а затем добавьте дополнительные кадры:

```csharp
var frames = LoadFrames(Path.Combine(dataDir, "Animation frames")).ToArray();

using (var image = new GifImage(new GifFrameBlock(frames[0])))
{
    for (var index = 1; index < frames.Length; index++)
    {
        image.AddPage(frames[index]);
    }
    
    image.Save(Path.Combine(outputDir, "Multipage.gif"));
}
```
Этот фрагмент загружает каждый кадр и добавляет его последовательно в GIF. `GifImage` конструктор инициализирует GIF, используя первый кадр.

#### Советы по устранению неполадок
- Убедитесь, что все рамы имеют одинаковые размеры.
- Проверьте пути к файлам на наличие опечаток или неверных ссылок на каталоги.
