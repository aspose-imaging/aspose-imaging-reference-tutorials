# Как загрузить и пересохранить изображения DICOM в BMP с помощью Aspose.Imaging Java

## Введение

В мире цифровой визуализации управление медицинскими изображениями имеет решающее значение, и **aspose imaging java** делает эту задачу гораздо проще. Независимо от того, нужно ли вам архивировать файлы DICOM, отображать их в веб‑портале или интегрировать в рабочий процесс здравоохранения, конвертация DICOM в BMP при сохранении качества — распространённое требование. В этом руководстве вы узнаете, как загрузить изображение DICOM, преобразовать его в BMP и пропорционально изменить высоту — всё с чистым, готовым к продакшну Java‑кодом.

**Что вы узнаете**

- Как конвертировать изображения DICOM в BMP с помощью **aspose imaging java**
- Техники изменения размера изображений DICOM с сохранением пропорций
- Настройка Aspose.Imaging для Java в вашей среде разработки

Прежде чем приступить к реализации, убедитесь, что у вас выполнены все предварительные требования.

## Быстрые ответы
- **Какая библиотека нужна?** Aspose.Imaging for Java (aspose imaging java)  
- **Можно ли конвертировать DICOM в BMP одной строкой?** Нет, сначала нужно загрузить изображение, затем сохранить его.  
- **Нужна ли лицензия для продакшна?** Да, требуется действующая лицензия Aspose.Imaging.  
- **Является ли изменение размера обязательным?** Нет, шаг изменения размера можно пропустить, если требуется только конвертация формата.  
- **Можно ли пакетно обрабатывать множество файлов?** Конечно — оберните тот же код в цикл или используйте Java‑streams.

Она абстрагирует низкоуровневые детали работы с изображениями, чтобы вы могли сосредоточиться на бизнес‑логике, а не на манипуляциях с пикселями.

## Руководство по реализации

Мы рассмотрим два практических сценария:

1. **Загрузить файл DICOM и сохранить его как BMP** (основная операция «save image bmp»).  
2. **Пропорционально изменить высоту изображения** перед сохранением, что полезно для веб‑миниатюр.

### Загрузка и пересохранение изображения DICOM в BMP

#### Обзор
Этот фрагмент демонстрирует минимальные шаги, необходимые для чтения файла DICOM и записи его в BMP.

#### Шаг за шагом

1. **Определите пути входного и выходного файлов** — подгоните их под вашу среду.  
2. **Загрузите изображение DICOM** с помощью `DicomImage`.  
3. **Сохраните его как BMP** с использованием стандартных `BmpOptions`.

```java
String dataDir = "YOUR_DOCUMENT_DIRECTORY";
String inputFile = dataDir + "image.dcm";
String outputFile = "YOUR_OUTPUT_DIRECTORY" + "ResizedOutput.bmp";

try (DicomImage image = (DicomImage) Image.load(inputFile)) {
    // Save the image as a BMP file.
    image.save(outputFile, new BmpOptions());
}
```

**Ключевые параметры**

- `inputFile`: полный путь к исходному файлу DICOM.  
- `outputFile`: путь к целевому BMP‑файлу.  
- `BmpOptions()`: использует настройки BMP по умолчанию; при необходимости можно настроить сжатие или формат пикселей.

### Пропорциональное изменение высоты

#### Обзор
Иногда требуется уменьшить изображение для более быстрой загрузки на веб‑странице. Ниже код изменяет высоту до 100 пикселей, сохраняя соотношение сторон.

#### Шаг за шагом

```java
String inputFile = dataDir + "image.dcm";
String outputFile = "YOUR_OUTPUT_DIRECTORY" + "ResizeHeightProportionally_out.bmp";

try (DicomImage image = (DicomImage) Image.load(inputFile)) {
    // Resize the height proportionally to 100 pixels.
    image.resizeHeightProportionally(100, ResizeType.AdaptiveResample);
    
    // Save the resized image in BMP format.
    image.save(outputFile, new BmpOptions());
}
```

**Важные детали**

- `resizeHeightProportionally(100, ResizeType.AdaptiveResample)` — первый аргумент задаёт целевую высоту в пикселях, второй указывает Aspose.Imaging использовать высококачественный адаптивный ресемплинг.  
- Метод автоматически вычисляет новую ширину, поэтому изображение никогда не будет растянуто.

## Распространённые проблемы и их решения

| Симптом | Возможная причина | Решение |
|---------|-------------------|---------|
| Ошибка `Unsupported format` | Используется старая версия Aspose.Imaging, не поддерживающая DICOM | Обновитесь до последней версии (≥ 25.5). |
| Исключение Out‑of‑memory при больших файлах DICOM | Изображение не освобождается или слишком велико для кучи | Увеличьте размер кучи JVM (`-Xmx2g`) и сохраняйте шаблон `try (DicomImage …)`. |
| BMP‑вывод чёрный или пустой | Файл DICOM содержит только метаданные (нет пиксельных данных) | Проверьте, что исходный DICOM содержит кадры; используйте `image.getFramesCount()` для проверки. |
| Изменённое изображение выглядит размытым | Используется низкокачественный тип изменения размера | Перейдите на `ResizeType.AdaptiveResample` (как в примере) или `ResizeType.HighQualityBicubic`. |

## Часто задаваемые вопросы

**В: В чём разница между `save image bmp` и `save image png`?**  
О: BMP — несжатый, без потерь формат, сохраняющий каждый пиксель, тогда как PNG применяет безпотерьное сжатие для уменьшения размера файла. Используйте BMP, когда нужна точная пиксельная достоверность.

**В: Можно ли конвертировать несколько файлов DICOM за один запуск?**  
О: Да, просто переберите каталог с файлами `.dcm` и примените ту же логику загрузки‑сохранения внутри цикла.

**В: Поддерживает ли aspose imaging java многокадровые серии DICOM?**  
О: Абсолютно — вы можете получить каждый кадр через `image.getFrames()` и сохранять их по отдельности или объединять в один BMP.

**В: Нужна ли лицензия для разработки?**  
О: Для оценки можно использовать бесплатную пробную или временную лицензию, но для продакшн‑развёртываний требуется приобретённая лицензия.

**В: Как работать с метаданными DICOM (имя пациента, ID исследования) после конвертации?**  
О: Aspose.Imaging позволяет читать теги DICOM через `image.getMetaData()`. Затем эту информацию можно внедрить в метаданные BMP или сохранить в отдельной базе данных.
