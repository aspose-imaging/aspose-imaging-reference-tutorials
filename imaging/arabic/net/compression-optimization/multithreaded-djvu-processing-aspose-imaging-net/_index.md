## دليل التنفيذ

### نظرة عامة على معالجة DJVU متعددة الخيوط

تتيح لك هذه الميزة معالجة صور متعددة من ملف DJVU في آنٍ واحد، باستخدام تعدد العمليات لتحسين سرعة المعالجة وكفاءتها. لنشرحها خطوة بخطوة.

#### الخطوة 1: تحديد دليل البيانات الخاص بك

قم بتعيين الدليل الذي سيتم تخزين ملفات DJVU فيه:

```csharp
string dataDir = "@YOUR_DOCUMENT_DIRECTORY"; 
```

#### الخطوة 2: تحديد ملف الإدخال والخيوط

قم بتحديد عدد الخيوط التي سيتم معالجتها، والتي يمكنك تعديلها استنادًا إلى إمكانيات جهازك:

```csharp
int numThreads = 20; // قم بتعديل هذا الرقم حسب الحاجة
```

#### الخطوة 3: إنشاء مهام للمعالجة المتزامنة

استخدام `Task.Run`إنشاء مهام لمعالجة أجزاء من ملف DJVU في وقت واحد. ستتولى كل مهمة جزءًا من عبء معالجة الصورة.

```csharp
var tasks = Enumerable.Range(1, numThreads).Select(taskNum =>
    Task.Run(() =>
    {
        string inputFile = Path.Combine(dataDir, "Sample.djvu");
        string outputFile = Path.Combine("@YOUR_OUTPUT_DIRECTORY", 
            $"Sample_task{taskNum}.png"); 

        using (FileStream fs = File.OpenRead(inputFile))
        {
            using (Image image = Image.Load(fs))
            {
                // احفظ كل صورة تمت معالجتها بصيغة PNG
                image.Save(outputFile, new PngOptions());
            }
        }
    }));
```

#### الخطوة 4: انتظر حتى اكتمال جميع المهام

تأكد من إكمال تنفيذ جميع الخيوط:

```csharp
Task.WaitAll(tasks.ToArray());
```

### خيارات تكوين المفاتيح ونصائح استكشاف الأخطاء وإصلاحها

- **عدد الخيوط:** تأكد من عدم تجاوز عدد نوى وحدة المعالجة المركزية لجهازك عند الإعداد `numThreads`.
- **معالجة الأخطاء:** قم بتنفيذ كتل try-catch داخل كل مهمة للتعامل مع الاستثناءات دون تعطل التطبيق بأكمله.
- **إدارة الموارد:** استخدم دائما `using` بيانات للتخلص من الموارد، والتأكد من إغلاق تدفقات الملفات بشكل صحيح.
