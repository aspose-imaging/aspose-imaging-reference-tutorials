## O que é java batch image processing?
Java batch image processing refere‑se ao manuseio automatizado de grandes grupos de imagens—carregamento, conversão, redimensionamento, filtragem ou análise—em uma única operação eficiente. ## Perguntas Frequentes

**Q: Posso usar Aspose.Imaging em projetos Java comerciais?**  
A: Sim. Uma licença comercial é necessária para uso em produção, e uma avaliação gratuita está disponível.

**Q: A biblioteca suporta as versões mais recentes do Java?**  
A: A biblioteca é totalmente compatível com Java 8, 11, 17 e versões LTS mais recentes.

**Q: Como o processamento em lote afeta o uso de memória?**  
A: Aspose.Imaging usa streaming e carregamento preguiçoso para manter a pegada de memória baixa; você também pode configurar limites de memória personalizados para lotes grandes.

**Q: É possível converter imagens sem gravá‑las em disco?**  
A: Absolutamente. Você pode trabalhar com streams (por exemplo, `ByteArrayInputStream`/`ByteArrayOutputStream`) para converter imagens inteiramente na memória.

**Q: Quais formatos são suportados para conversão DICOM?**  
A: Todas as modalidades DICOM padrão são suportadas, incluindo JPEG‑Lossless, JPEG‑2000 e dados de pixel brutos.
