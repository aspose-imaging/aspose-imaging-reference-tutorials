## Frequently Asked Questions

**Q: Can I use Aspose.Imaging for commercial Java projects?**  
A: Yes. A commercial license is required for production use, and a free trial is available for evaluation.

**Q: Does the library support the latest Java versions?**  
A: The library is fully compatible with Java 8, 11, 17, and newer LTS releases.

**Q: How does batch processing affect memory usage?**  
A: Aspose.Imaging uses streaming and lazy loading to keep memory footprints low; you can also configure custom memory limits for large batches.

**Q: Is it possible to convert images without writing them to disk?**  
A: Absolutely. You can work with streams (e.g., `ByteArrayInputStream`/`ByteArrayOutputStream`) to convert images entirely in memory.

**Q: What formats are supported for DICOM conversion?**  
A: All standard DICOM modalities are supported, including JPEG‑Lossless, JPEG‑2000, and raw pixel data.
