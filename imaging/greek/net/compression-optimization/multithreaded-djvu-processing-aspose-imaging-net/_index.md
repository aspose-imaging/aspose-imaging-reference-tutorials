## Οδηγός Εφαρμογής

### Επισκόπηση της επεξεργασίας πολυνηματικού DJVU

Αυτή η λειτουργία σάς επιτρέπει να επεξεργάζεστε πολλαπλές εικόνες από ένα αρχείο DJVU ταυτόχρονα, χρησιμοποιώντας πολλαπλά νήματα για να βελτιώσετε την ταχύτητα και την αποτελεσματικότητα της επεξεργασίας. Ας το αναλύσουμε βήμα προς βήμα.

#### Βήμα 1: Ορίστε τον κατάλογο δεδομένων σας

Ορίστε τον κατάλογο όπου αποθηκεύονται τα αρχεία DJVU:

```csharp
string dataDir = "@YOUR_DOCUMENT_DIRECTORY"; 
```

#### Βήμα 2: Καθορίστε το αρχείο εισόδου και τα νήματα

Ορίστε τον αριθμό των νημάτων για επεξεργασία, τον οποίο μπορείτε να προσαρμόσετε με βάση τις δυνατότητες του μηχανήματός σας:

```csharp
int numThreads = 20; // Προσαρμόστε αυτόν τον αριθμό όπως απαιτείται
```

#### Βήμα 3: Δημιουργία εργασιών για ταυτόχρονη επεξεργασία

Χρησιμοποιώντας `Task.Run`, δημιουργήστε εργασίες που επεξεργάζονται τμήματα του αρχείου DJVU ταυτόχρονα. Κάθε εργασία θα χειρίζεται ένα μέρος του φόρτου εργασίας επεξεργασίας εικόνας.

```csharp
var tasks = Enumerable.Range(1, numThreads).Select(taskNum =>
    Task.Run(() =>
    {
        string inputFile = Path.Combine(dataDir, "Sample.djvu");
        string outputFile = Path.Combine("@YOUR_OUTPUT_DIRECTORY", 
            $"Sample_task{taskNum}.png"); 

        using (FileStream fs = File.OpenRead(inputFile))
        {
            using (Image image = Image.Load(fs))
            {
                // Αποθηκεύστε κάθε επεξεργασμένη εικόνα ως PNG
                image.Save(outputFile, new PngOptions());
            }
        }
    }));
```

#### Βήμα 4: Περιμένετε να ολοκληρωθούν όλες οι εργασίες

Βεβαιωθείτε ότι όλα τα νήματα ολοκληρώνουν την εκτέλεσή τους:

```csharp
Task.WaitAll(tasks.ToArray());
```

### Βασικές επιλογές διαμόρφωσης και συμβουλές αντιμετώπισης προβλημάτων

- **Αριθμός νημάτων:** Βεβαιωθείτε ότι δεν υπερβαίνετε τον αριθμό πυρήνων CPU του μηχανήματός σας κατά τη ρύθμιση `numThreads`.
- **Χειρισμός σφαλμάτων:** Υλοποιήστε μπλοκ try-catch σε κάθε εργασία για να χειρίζεστε εξαιρέσεις χωρίς να καταρρέει ολόκληρη η εφαρμογή.
- **Διαχείριση Πόρων:** Να χρησιμοποιείτε πάντα `using` δηλώσεις για την απόρριψη πόρων, διασφαλίζοντας ότι οι ροές αρχείων κλείνουν σωστά.
