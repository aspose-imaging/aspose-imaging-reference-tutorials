## Φόρτωση Εικόνας

### Επισκόπηση
Η φόρτωση μιας εικόνας WMF είναι το πρώτο βήμα πριν από οποιαδήποτε επεξεργασία ή μετατροπή. Θα χρησιμοποιήσουμε την κλάση `Image` του Aspose.Imaging για αυτό το σκοπό.

### Βήματα Υλοποίησης

**1. Εισαγωγή Απαραίτητων Κλάσεων**

Η κλάση `Image` είναι το κορυφαίο αντικείμενο του Aspose.Imaging που αντιπροσωπεύει ένα μοναδικό αρχείο εικόνας στη μνήμη. Παρέχει μεθόδους για φόρτωση, αποθήκευση και πρόσβαση σε μεταδεδομένα εικόνας.
```java
import com.aspose.imaging.Image;
```

**2. Φόρτωση της Εικόνας WMF**

Χρησιμοποιήστε τη μέθοδο `Image.load()` για να διαβάσετε το αρχείο WMF από έναν καθορισμένο φάκελο. Αυτή η κλήση επιστρέφει ένα αντικείμενο `Image` που μπορείτε να περάσετε στις επιλογές rasterization αργότερα.
```java
try (Image image = Image.load("YOUR_DOCUMENT_DIRECTORY/input.wmf")) {
    // Further processing can be done here.
}
```

*Επεξήγηση*: Η μέθοδος `Image.load()` ανοίγει το καθορισμένο αρχείο εικόνας και επιστρέφει ένα αντικείμενο `Image`, επιτρέποντάς σας να εκτελέσετε περαιτέρω λειτουργίες όπως μετατροπή ή επεξεργασία.

## Ορισμός Ρυθμίσεων Rasterization

### Επισκόπηση
Οι ρυθμίσεις rasterization ορίζουν πώς η εικόνα WMF σας μετατρέπεται σε raster μορφή πριν ενσωματωθεί στο τελικό SVG. Αυτές οι ρυθμίσεις εξασφαλίζουν ότι το SVG εξόδου διατηρεί την επιθυμητή ποιότητα και διαστάσεις.

### Βήματα Υλοποίησης

**1. Εισαγωγή Απαραίτητων Κλάσεων**

`WmfRasterizationOptions` είναι η κλάση που ελέγχει το μέγεθος σελίδας, το χρώμα φόντου και άλλες παραμέτρους απόδοσης για τη μετατροπή WMF‑σε‑SVG.
```java
import com.aspose.imaging.imageoptions.WmfRasterizationOptions;
```

**2. Διαμόρφωση Ρυθμίσεων Rasterization**

Δημιουργήστε μια παρουσία του `WmfRasterizationOptions` για να ορίσετε το πλάτος και το ύψος της σελίδας:
```java
WmfRasterizationOptions options = new WmfRasterizationOptions();
options.setPageWidth(800); // Set the desired output image width.
options.setPageHeight(600); // Set the desired output image height.
```

*Επεξήγηση*: Ορίζοντας το `pageWidth` και το `pageHeight` εξασφαλίζετε ότι το SVG σας διατηρεί σταθερές διαστάσεις κατά τη μετατροπή.

## Αποθήκευση Εικόνας ως SVG

### Επισκόπηση
Το τελικό βήμα περιλαμβάνει την αποθήκευση της επεξεργασμένης εικόνας WMF ως αρχείο SVG. Εδώ εφαρμόζονται όλες οι προηγούμενες ρυθμίσεις για την παραγωγή ενός υψηλής ποιότητας διανύσματος.

### Βήματα Υλοποίησης

**1. Εισαγωγή Απαραίτητων Κλάσεων**

`SvgOptions` είναι η κλάση που συγκεντρώνει ρυθμίσεις ειδικές για SVG, συμπεριλαμβανομένων των επιλογών rasterization που ορίσατε προηγουμένως.
```java
import com.aspose.imaging.imageoptions.SvgOptions;
```

**2. Μετατροπή και Αποθήκευση ως SVG**

Χρησιμοποιήστε τη μέθοδο `save()` με `SvgOptions` για να αποθηκεύσετε την εικόνα σας σε μορφή SVG:
```java
image.save("YOUR_OUTPUT_DIRECTORY/ConvertWMFMetaFileToSVG_out.svg", new SvgOptions() {
{
    setVectorRasterizationOptions(options);
}
});
```

*Επεξήγηση*: Η κλάση `SvgOptions` σας επιτρέπει να ορίσετε τις ρυθμίσεις rasterization για διανυσματικές εικόνες. Ορίζοντας το `vectorRasterizationOptions`, διασφαλίζουμε ότι η έξοδος SVG ακολουθεί τις καθορισμένες διαστάσεις.

## Πώς να Μετατρέψετε WMF σε SVG σε Java;

Φορτώστε το αρχείο WMF με `Image.load("input.wmf")`, διαμορφώστε ένα αντικείμενο `WmfRasterizationOptions` με το επιθυμητό πλάτος και ύψος, τυλίξτε το σε μια παρουσία `SvgOptions` και τέλος καλέστε `image.save("output.svg", svgOptions)`. Αυτή η ροή τεσσάρων βημάτων διαχειρίζεται την πιστότητα του διανύσματος, το φόντο και το προαιρετικό scaling αυτόματα, παρέχοντας ένα καθαρό SVG έτοιμο για web ή εκτύπωση.

## Συχνά Προβλήματα και Λύσεις

- **FileNotFoundException** – Ελέγξτε ξανά τη διαδρομή του αρχείου WMF και βεβαιωθείτε ότι το αρχείο υπάρχει.
- **Missing Output Directory** – Δημιουργήστε το φάκελο προορισμού πριν καλέσετε `save()`.
- **Unexpected SVG Size** – Προσαρμόστε το `pageWidth`/`pageHeight` στο `WmfRasterizationOptions` ή ορίστε `scale` για λεπτομερή ρύθμιση διαστάσεων.
- **Memory Errors** – Χρησιμοποιήστε try‑with‑resources για αυτόματη διαγραφή του αντικειμένου `Image` και σκεφτείτε την αύξηση του heap της JVM (`-Xmx`) για πολύ μεγάλα αρχεία.
