# Πώς να φορτώσετε και να αποθηκεύσετε ξανά εικόνες DICOM ως BMP χρησιμοποιώντας το Aspose.Imaging Java

## Εισαγωγή

Στον κόσμο της ψηφιακής απεικόνισης, η διαχείριση ιατρικών εικόνων είναι κρίσιμη, και **aspose imaging java** κάνει τη δουλειά πολύ πιο εύκολη. Είτε χρειάζεστε να αρχειοθετήσετε αρχεία DICOM, να τα εμφανίσετε σε μια διαδικτυακή πύλη, είτε να τα ενσωματώσετε σε μια ροή εργασίας υγειονομικής περίθαλψης, η μετατροπή DICOM σε BMP διατηρώντας την ποιότητα είναι κοινή απαίτηση. Σε αυτό το tutorial θα μάθετε πώς να φορτώσετε μια εικόνα DICOM, να τη μετατρέψετε σε BMP και να αλλάξετε το ύψος της αναλογικά — όλα με καθαρό, έτοιμο για παραγωγή κώδικα Java.

**Τι θα μάθετε**

- Πώς να μετατρέψετε εικόνες DICOM σε BMP χρησιμοποιώντας **aspose imaging java**
- Τεχνικές για αλλαγή μεγέθους εικόνων DICOM διατηρώντας τις αναλογίες
- Ρύθμιση του Aspose.Imaging για Java στο περιβάλλον ανάπτυξής σας

Πριν βυθιστείτε στην υλοποίηση, ας βεβαιωθούμε ότι έχετε καλύψει τις προαπαιτούμενες απαιτήσεις.

## Οδηγός Υλοποίησης

Θα εξετάσουμε δύο πρακτικά σενάρια:

1. **Φόρτωση ενός αρχείου DICOM και αποθήκευση του ως BMP** (η βασική λειτουργία «save image bmp»).  
2. **Αλλαγή του ύψους της εικόνας αναλογικά** πριν την αποθήκευση, χρήσιμη για μικρογραφίες ιστού.

### Φόρτωση και επανααποθήκευση εικόνας DICOM ως BMP

#### Επισκόπηση
Αυτό το απόσπασμα κώδικα δείχνει τα ελάχιστα βήματα για ανάγνωση ενός αρχείου DICOM και εγγραφή του ως αρχείο BMP.

#### Βήμα‑βήμα

1. **Ορίστε διαδρομές εισόδου και εξόδου** – προσαρμόστε τις ώστε να ταιριάζουν με το περιβάλλον σας.  
2. **Φορτώστε την εικόνα DICOM** χρησιμοποιώντας `DicomImage`.  
3. **Αποθηκεύστε την ως BMP** με τις προεπιλεγμένες `BmpOptions`.

```java
String dataDir = "YOUR_DOCUMENT_DIRECTORY";
String inputFile = dataDir + "image.dcm";
String outputFile = "YOUR_OUTPUT_DIRECTORY" + "ResizedOutput.bmp";

try (DicomImage image = (DicomImage) Image.load(inputFile)) {
    // Save the image as a BMP file.
    image.save(outputFile, new BmpOptions());
}
```

**Κύριες Παράμετροι**

- `inputFile`: Πλήρης διαδρομή προς το πηγαίο αρχείο DICOM.  
- `outputFile`: Διαδρομή προορισμού για το αρχείο BMP.  
- `BmpOptions()`: Χρησιμοποιεί τις προεπιλεγμένες ρυθμίσεις BMP· μπορείτε να προσαρμόσετε τη συμπίεση ή τη μορφή εικονοστοιχείου αν χρειάζεται.

### Αλλαγή ύψους αναλογικά

#### Επισκόπηση
Μερικές φορές χρειάζεστε μικρότερη εικόνα για ταχύτερη φόρτωση σε μια ιστοσελίδα. Ο παρακάτω κώδικας αλλάζει το ύψος σε 100 pixel διατηρώντας την αναλογία διαστάσεων.

#### Βήμα‑βήμα

```java
String inputFile = dataDir + "image.dcm";
String outputFile = "YOUR_OUTPUT_DIRECTORY" + "ResizeHeightProportionally_out.bmp";

try (DicomImage image = (DicomImage) Image.load(inputFile)) {
    // Resize the height proportionally to 100 pixels.
    image.resizeHeightProportionally(100, ResizeType.AdaptiveResample);
    
    // Save the resized image in BMP format.
    image.save(outputFile, new BmpOptions());
}
```

**Σημαντικές λεπτομέρειες**

- `resizeHeightProportionally(100, ResizeType.AdaptiveResample)` – το πρώτο όρισμα είναι το επιθυμητό ύψος σε εικονοστοιχεία· το δεύτερο όρισμα λέει στο Aspose.Imaging να χρησιμοποιήσει προσαρμοστική επαναδειγματοληψία υψηλής ποιότητας.  
- Η μέθοδος υπολογίζει αυτόματα το νέο πλάτος, ώστε η εικόνα να μην φαίνεται τεντωμένη.

## Παράγοντες Απόδοσης

- **Αλλαγή μεγέθους πριν τη μετατροπή**: Η μείωση των διαστάσεων νωρίς μειώνει τη χρήση μνήμης και επιταχύνει την αποθήκευση.  
- **Χρησιμοποιήστε try‑with‑resources** (όπως φαίνεται) για να διασφαλίσετε ότι η εικόνα αποδεσμεύεται άμεσα, αποτρέποντας διαρροές μνήμης.  
- **Λειτουργία Μαζικής Επεξεργασίας**: Για μεγάλα όγκους, επεξεργαστείτε αρχεία παράλληλα χρησιμοποιώντας `ExecutorService`, αλλά παρακολουθήστε το μέγεθος της στοίβας.

## Κοινά Προβλήματα και Λύσεις

| Σύμπτωμα | Πιθανή Αιτία | Διόρθωση |
|----------|--------------|----------|
| `Unsupported format` error | Χρήση παλιάς έκδοσης Aspose.Imaging που δεν υποστηρίζει DICOM | Αναβαθμίστε στην τελευταία έκδοση (≥ 25.5). |
| Εξαίρεση έλλειψης μνήμης σε μεγάλα αρχεία DICOM | Η εικόνα δεν αποδεσμεύεται ή είναι πολύ μεγάλη για τη στοίβα | Αυξήστε τη στοίβα JVM (`-Xmx2g`) και τηρείτε το πρότυπο `try (DicomImage …)` . |
| Η έξοδος BMP είναι μαύρη ή κενή | Το αρχείο DICOM περιέχει μόνο μεταδεδομένα (χωρίς εικονοστοιχεία) | Επαληθεύστε ότι το πηγαίο DICOM περιέχει καρέ εικόνας· χρησιμοποιήστε `image.getFramesCount()` για έλεγχο. |
| Η αλλαγμένη εικόνα φαίνεται θολή | Χρήση τύπου αλλαγής μεγέθους χαμηλής ποιότητας | Αλλάξτε σε `ResizeType.AdaptiveResample` (όπως στο παράδειγμα) ή `ResizeType.HighQualityBicubic`. |

## Συχνές Ερωτήσεις

**Ε: Ποια είναι η διαφορά μεταξύ `save image bmp` και `save image png`;**  
Α: Το BMP είναι μια ασυμπίεστη, χωρίς απώλειες μορφή που διατηρεί κάθε εικονοστοιχείο, ενώ το PNG εφαρμόζει ασυμπίεστη συμπίεση για μείωση του μεγέθους του αρχείου. Χρησιμοποιήστε BMP όταν χρειάζεστε ακριβή πιστότητα εικονοστοιχείων.

**Ε: Μπορώ να μετατρέψω πολλαπλά αρχεία DICOM σε μία εκτέλεση;**  
Α: Ναι, απλώς επαναλάβετε τη λογική φόρτωσης‑αποθήκευσης για κάθε αρχείο `.dcm` μέσα σε βρόχο.

**Ε: Υποστηρίζει το aspose imaging java σειρές DICOM πολλαπλών καρέ;**  
Α: Απόλυτα – μπορείτε να προσπελάσετε κάθε καρέ μέσω `image.getFrames()` και να τα αποθηκεύσετε ξεχωριστά ή να τα συνδυάσετε σε ένα BMP.

**Ε: Χρειάζεται άδεια για ανάπτυξη;**  
Α: Μπορείτε να χρησιμοποιήσετε δωρεάν δοκιμή ή προσωρινή άδεια για αξιολόγηση, αλλά απαιτείται αγορασμένη άδεια για παραγωγικές εγκαταστάσεις.

**Ε: Πώς διαχειρίζομαι τα μεταδεδομένα DICOM (όνομα ασθενούς, ID μελέτης) μετά τη μετατροπή;**  
Α: Το Aspose.Imaging σας επιτρέπει να διαβάζετε ετικέτες DICOM μέσω `image.getMetaData()`. Μπορείτε στη συνέχεια να ενσωματώσετε αυτές τις πληροφορίες στα μεταδεδομένα BMP ή σε ξεχωριστή βάση δεδομένων.
